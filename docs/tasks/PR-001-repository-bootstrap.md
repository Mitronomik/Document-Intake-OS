# PR-001 — Repository bootstrap

## Goal

Create a production-oriented Python 3.12 repository skeleton for the offline Document Intake OS Windows desktop application.

## Required reading

- `AGENTS.md`
- `docs/technical-specification.md`
- `docs/architecture.md`
- `docs/domain-model.md`
- `docs/security.md`
- `docs/testing-strategy.md`
- `docs/progress.md`

## Scope

- configure `pyproject.toml`;
- use uv for dependency management;
- commit `uv.lock` generated by uv;
- create the package layout defined in the architecture;
- configure pytest, Ruff and mypy;
- create a minimal PySide6 application entry point;
- create minimal automated smoke tests that use real PySide6 and do not skip Qt;
- verify the real Qt event loop starts and exits in offscreen mode;
- add CI checks that do not require real documents, terminal templates or OCR models;
- run CI on Ubuntu and Windows;
- update README development commands;
- update `docs/progress.md` without claiming completion before human acceptance.

## Hard constraints

- do not implement business functionality;
- do not add OCR;
- do not add database code;
- do not add network dependencies or telemetry;
- do not modify the supplied Excel templates;
- do not commit terminal templates or template-derived golden Excel files while the repository is public;
- do not include any real personal data;
- do not make new security or persistence decisions beyond the temporary public-repository decision recorded for PR-001.

## Non-goals

- image processing;
- domain implementation;
- persistence;
- Excel export;
- installer;
- production authentication.

## Required verification commands

```bash
uv sync --locked --all-extras --dev
uv run ruff check .
uv run ruff format --check .
uv run mypy src
uv run pytest -ra
uv build
```

## Acceptance criteria

1. `.python-version` contains `3.12`.
2. `requires-python` is `>=3.12,<3.13`.
3. `uv.lock` is generated by uv and committed.
4. `uv sync --locked --all-extras --dev` succeeds.
5. Hatchling builds the src-layout package and explicitly packages `src/document_intake`.
6. `uv build` succeeds.
7. `uv run ruff check .` succeeds.
8. `uv run ruff format --check .` succeeds.
9. `uv run mypy src` succeeds.
10. `uv run pytest -ra` succeeds.
11. Qt tests do not skip.
12. The real Qt event loop starts and exits in offscreen mode.
13. The package can be imported.
14. The minimal desktop entry point starts without network access.
15. README contains locked local development commands.
16. Ubuntu CI passes.
17. Windows CI passes.
18. CI uses synthetic/no-document fixtures only.
19. No terminal templates or template-derived golden Excel files are committed.
20. No personal data are committed.
21. `docs/progress.md` records `PR-001 IN REVIEW` while implementation is submitted but not accepted.

## Progress status rule

While the implementation is submitted but not yet accepted, `docs/progress.md` records `PR-001 IN REVIEW`.

PR-001 may be marked completed only after all acceptance criteria pass, human review is completed and the PR is merged.
